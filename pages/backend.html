Back-End

O que é Back-End?

O Back-End é a parte de um sistema que o usuário não vê diretamente. Ele é responsável pela lógica, regras de negócio, armazenamento e segurança. Em resumo, é no Back-End que os dados são processados e armazenados.

Principais responsabilidades:

  • Gerenciar bancos de dados e informações;
  • Criar APIs para comunicação com o Front-End;
  • Implementar regras de negócio e autenticação de usuários;
  • Garantir a segurança e escalabilidade das aplicações.

Principais tecnologias:

  • Node.js: execução de JavaScript no servidor.
  • Python (Django/Flask): frameworks populares e robustos.
  • Java (Spring Boot): usado em sistemas corporativos.
  • PHP (Laravel): muito usado em sites e aplicações web.
  • Bancos de Dados: MySQL, PostgreSQL, MongoDB, Redis.

Resumo : Back-End = tudo que acontece “por trás das câmeras” para que o sistema funcione.

Comandos e Conceitos Básicos de Back-End

Node.js (JavaScript no servidor)

ComandoPara que serve
npm initInicia um projeto Node.js.
npm install expressInstala o framework Express.
const express = require("express")Importa o Express no código.
app.get("/", ...)Cria uma rota de acesso.
app.listen(3000)Faz o servidor rodar na porta 3000.

Python (Flask/Django)

ComandoPara que serve
pip install flaskInstala o framework Flask.
from flask import FlaskImporta a biblioteca Flask.
@app.route("/")Cria uma rota no servidor.
python manage.py runserverExecuta o servidor no Django.

Banco de Dados (SQL)

ComandoPara que serve
CREATE DATABASE nome;Cria um banco de dados.
CREATE TABLE tabela (...);Cria uma tabela.
SELECT * FROM tabela;Consulta todos os registros.
INSERT INTO tabela VALUES (...);Insere um registro.
UPDATE tabela SET coluna=valor;Atualiza dados existentes.
DELETE FROM tabela WHERE ...;Remove registros.

ConceitoPara que serve

MiddlewareFunções que interceptam requisições e respostas no servidor.
DeployColocar o sistema no ar (servidor ou nuvem).
